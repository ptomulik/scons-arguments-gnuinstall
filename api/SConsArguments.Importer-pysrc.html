  1  """`SConsArguments.Importer` 
  2   
  3  Importing SCons arguments (declarations) from module files. The idea of 
  4  importing argument declarations from python modules is similar to that of 
  5  loading SCons tools from python modules. A python file placed in appropriate 
  6  directory can act as a source of arguments' declarations. The default list of 
  7  directories that are looked up for arguments' modules is called ``argpath`` 
  8  and is returned by `GetDefaultArgpath()`. The list is constructed similarly to 
  9  the tools search path. In the default ``argpath`` we use ``"site_arguments"`` 
 10  instead of ``"site_tools"`` in the last component of a directory name. This 
 11  means that, in typical situation, argument modules will be placed in 
 12  ``"site_scons/site_arguments/"``. Only existing directories get included in the 
 13  default ``argpath``. Several modules are also included in the 
 14  ``SConsArguments`` package. 
 15   
 16  A module containing argument declarations is a python module with the 
 17  following function defined:: 
 18   
 19      def arguments(**kw) 
 20   
 21  This function shall return a dict with argument names as keys and argument 
 22  declarations as values. Each declaration should have a form of dict with keys 
 23  and values specified by the documentation of 
 24  `SConsArguments.Declaration.DeclareArgument`. For example:: 
 25   
 26      # site_scons/site_arguments/mine.py 
 27      def arguments(**kw): 
 28          return { 
 29              'arg1' : { 'help' : 'This is arg1' }, 
 30              'arg2' : { 'help' : 'This is arg2' } 
 31          } 
 32   
 33  This function may use the utility function `export_arguments()` to handle 
 34  common parameteres. See the source code of existing modules in 
 35  ``"SConsArguments"`` to find an example. 
 36   
 37  Modules may be imported with `ImportArguments()` function, which takes a 
 38  module name(s) as firsta argument (so, `ImportArguments("mine")` for the above 
 39  example).
