Class Hierarchy

  • object: The most base type
    • SConsArguments.Declaration._ArgumentDeclaration: Declaration of single argument.
    • SConsArguments.NameConv._ArgumentNameConv: Provides a systematic way to transform argument names into keys identifying their endpoints.
    • SConsArguments.Arguments._Arguments: Binds arguments to their endpoints (construction variables, command-line variables and command-line options).
    • SConsArguments.Proxy._ArgumentsProxy: Proxy object used to access indirectly variables stored in a target dictionary. The indirection layer automatically maps variable names between two namespaces (user and target).
    • SConsArguments.VariablesWrapper._VariablesWrapper: Wrapper class used to overcome several issues with original implementation of SCons Variables.
    • SConsArguments.Util._missing: Represents missing argument to function.
    • SConsArguments.Util._notfound: Something that has not been found, a result of failed search.
    • SConsArguments.Util._undef: Represents undefined/inexistent variable. This is not the same as None
    • dict: dict() -> new empty dictionary dict(mapping) -> new dictionary initialized from a mapping object's (key, value) pairs dict(iterable) -> new dictionary initialized as if via: d = {} for k, v in iterable: d[k] = v dict(**kwargs) -> new dictionary initialized with the name=value pairs in the keyword argument list.
      • SConsArguments.Declarations._ArgumentDeclarations: Dict-like object with argument declarations as values.
    • type: type(object) -> the object's type type(name, bases, dict) -> a new type
      • SConsArguments.Util._missing_meta: Meta-class for the _missing class
      • SConsArguments.Util._notfound_meta: Meta-class for the _notfound class
      • SConsArguments.Util._undef_meta: Meta-class for the _undef class
